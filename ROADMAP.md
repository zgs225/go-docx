# go-docx 12 个月路线图（2026）

## 背景与目标

### 库定位
`go-docx` 是一个面向 Go 的 `.docx` 读写库，目标是在开源场景下提供可控、可扩展、可自动化的 Word 文档处理能力，覆盖“解析、编辑、生成、回写”全链路。

### 目标用户
- 需要批量生成合同/报告/通知等文档的后端服务团队
- 需要在保留原文档结构前提下进行自动化修改的文档平台团队
- 需要在 CI/服务端环境执行文档处理流水线的工程团队

### 12 个月目标
- 建立“高保真回写”能力：常见复杂文档在 parse/write 后不丢结构
- 完成高频业务能力闭环：模板替换、页眉页脚页码、表格高级操作
- 建立样式/编号/引用等文档体系化能力
- 为 SDT、图表/公式与大文档性能优化打下可迭代基础

## 当前能力盘点

### 已支持能力
- 文档解析与打包回写
- 段落与 run 级文本编辑（颜色、字号、下划线、高亮、字体等）
- 超链接编辑
- 图片（inline/anchor）插入与基础尺寸控制
- 表格创建与基础设置（宽度、对齐、底纹、padding、部分边框属性）
- 形状、画布、分组相关结构的解析与部分生成能力
- 文档切分、段落/图形元素过滤等工具能力

### 主要缺口
- 缺少跨 run 的文本替换/占位符替换 API
- 缺少页眉/页脚/页码的统一读写入口
- 表格高级能力不足（横向合并、重复表头、布局控制统一 API）
- 缺少样式体系与编号体系的管理 API（style/numbering）
- 缺少脚注/尾注/批注/书签与域等引用体系能力
- 缺少 SDT（内容控件）读写与数据填充能力
- 缺少图表/公式等高级对象的可控策略（至少“保留 + 有限编辑”）

### 兼容性风险（重点）
- 解析层当前对部分未知节点以跳过方式处理，可能导致回写后结构信息丢失
- 对复杂真实文档（目录、域、引用、控件、修订）缺少系统化 round-trip 基线
- 一旦没有“保真底座”，后续新增功能会放大文档损坏风险

## docx 功能全景与高频排序

### 排序规则
`Priority Score = 频率(40%) + 价值(40%) + 落地可行性(20%)`

评分说明：
- 频率：业务中出现频率（1-5）
- 价值：对业务结果与工程成本影响（1-5）
- 落地可行性：在当前代码基线下的实现可达性（1-5）
- 最终分数：`(频率*0.4 + 价值*0.4 + 可行性*0.2) * 20`，满分 100

| 2 | 模板文本替换（含跨 run） | 缺失 | 5 | 5 | 4 | 96 |
| 3 | 页眉/页脚/页码 | 缺失 | 5 | 4 | 4 | 88 |
| 4 | 表格高级能力（合并/边框/布局） | 部分支持 | 4 | 5 | 4 | 88 |

| 6 | 引用体系（脚注/尾注/批注/书签/域） | 缺失 | 3 | 5 | 3 | 76 |

### Top 优先级列表
1. 文档保真回写（P0）
2. 模板文本替换（P0）
3. 页眉/页脚/页码（P0）
4. 表格高级能力（P1）
5. 样式与编号管理（P1）

## 12 个月分阶段计划（阶段 + 验收标准）

> 时间划分建议：Phase 0（Q1）、Phase 1（Q2）、Phase 2（Q3）、Phase 3（Q4）

### Phase 0：保真与稳定性底座（Q1）

阶段目标：
- 建立“复杂文档不破坏”的最小可信回写底座

关键交付物：
- 未知节点保留机制（至少覆盖 body/paragraph/run/table 常见层级）
- Golden round-trip 测试集（真实样本 + 预期校验）
- 回写一致性检查工具（结构差异与关键节点丢失检测）

验收标准：
- 指定样本文档执行 parse + write 后，关键节点丢失率为 0
- round-trip 用例通过率 >= 95%
- 对已有 API 场景无回归（现有测试全通过）

### Phase 1：高频业务能力（Q2）

阶段目标：
- 完成“模板替换 + 页眉页脚 + 表格增强”的高频能力闭环

关键交付物：
- `ReplaceText/ReplacePlaceholder`（支持跨 run）
- Header/Footer 读写入口与页码字段插入能力
- 表格增强第一批：`ColSpan`、统一边框 API、行高与重复表头能力

验收标准：
- 占位符替换在跨 run、含样式 run 场景正确率 >= 99%
- 页眉页脚在新建与解析后文档均可稳定读写
- 表格增强功能具备独立单元测试与端到端样例

### Phase 2：文档体系能力（Q3）

阶段目标：
- 建立样式、编号、引用体系的可编程入口

关键交付物：
- style 管理 API（读取/应用/新增基础样式）
- numbering 管理 API（abstractNum/num 定义与复用）
- 引用体系第一批：脚注、尾注、批注、书签/基础域

验收标准：
- 样式/编号在 Word 中显示与预期一致，且回写可复用
- 脚注/尾注/批注可创建、可解析、可回写
- 新增能力具有完整示例与最小迁移说明

### Phase 3：高级对象与性能（Q4）

阶段目标：
- 具备高级对象策略能力，并改善大文档可用性

关键交付物：
- SDT（内容控件）读写与基础填充
- 图表/公式的“保留 + 有限编辑”策略（优先不破坏）
- 大文档性能优化（内存峰值、处理吞吐、稳定性）

验收标准：
- SDT 在主流样本上可稳定提取/填充
- 含图表/公式文档 round-trip 不损坏关键结构
- 在基准样本上内存峰值/耗时相较基线显著改进（目标值在实施前冻结）

## 拟新增公共 API 草案（接口级别）

> 以下为规划草案，仅定义方向与入口，不代表最终签名。

### 文本
- `ReplaceText(find, replace string, opts ...ReplaceOption) error`
- `ReplacePlaceholder(data map[string]string, opts ...ReplaceOption) error`
- 关键能力：跨 run 合并匹配、替换后样式保持策略可配置

### 页面（Header/Footer）
- `GetHeader(kind HeaderKind) (*Header, error)`
- `GetFooter(kind FooterKind) (*Footer, error)`
- `SetHeader(kind HeaderKind, header *Header) error`
- `SetFooter(kind FooterKind, footer *Footer) error`
- `AddPageNumber(style PageNumberStyle) error`

### 表格
- `SetColSpan(row, col, span int) error`
- `SetRowSpan(row, col, span int) error`
- `SetCellBorder(row, col int, border CellBorder) error`
- `SetRepeatHeader(row int, enable bool) error`

### 样式与编号
- `ListStyles() []StyleInfo`
- `EnsureStyle(style StyleDef) (styleID string, err error)`
- `EnsureNumbering(def NumberingDef) (numID string, err error)`
- `ApplyParagraphStyle(p *Paragraph, styleID string) error`

## 验收标准（全局）

- 文档完整性：必须覆盖“功能全景、排序、阶段计划、验收标准、API 草案”
- 一致性：高优先级能力必须落在前两阶段（Phase 0/1）
- 可执行性：每阶段均包含“目标、关键交付物、验收标准”
- 变更范围：路线图阶段仅引入文档变更，不包含实现代码提交

## 风险与依赖

### 关键风险
- OOXML 细节复杂，且不同 Word 版本存在兼容差异
- 缺少真实业务样本时，测试通过不代表生产安全
- 回写保真与功能扩展可能存在结构冲突

### 外部依赖
- 稳定的真实 docx 样本库（含目录、域、页眉页脚、引用、控件、图表）
- CI 中可重复执行的 round-trip 对比工具链
- 明确的向后兼容策略（旧 API、旧文档行为）

### 向后兼容策略
- 新 API 以增量方式引入，避免破坏现有入口
- 对已支持功能保持默认行为不变，新增能力通过选项启用
- 关键行为变更需提供迁移说明与版本标记

## 附录

### 术语
- Round-trip：文档 `parse -> modify(可选) -> write` 后结构与关键语义保持一致
- 保真：回写后不丢关键节点、不破坏可编辑性与显示语义
- SDT：Structured Document Tag，Word 内容控件

### 优先级评分规则
- Score = `(频率*0.4 + 价值*0.4 + 可行性*0.2) * 20`
- 分数区间解释：
  - 90-100：必须立即推进（P0）
  - 80-89：高优先级（P1）
  - 65-79：中优先级（P2）
  - <65：后续规划（P3）
